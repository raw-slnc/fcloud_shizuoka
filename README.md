# Fcloud Shizuoka

A QGIS plugin that links a forest register GeoPackage with the Shizuoka Prefecture Forest Cloud public API, letting you browse forest stand attributes and cloud-side administrative records from a single dock panel.

This plugin is designed for GeoPackage data generated by **Shinrinbo Code Converter** and is intended for **Shizuoka Prefecture workflows only**.

## Features

- **Layer connection**: Detects loaded `.gpkg` and `.shp` layers that contain a `KEY1` field in the current QGIS project and lets you choose the target layer from a combo box
- **Selected stand viewer**: Shows the attributes of the currently selected forest stand in the left panel
- **Protective forest register search**: Search by city, Oaza, and parcel number, then highlight matching features on the map
- **Mori no Chikara search**: Filter records by forestry office, fiscal year, project type, and operator; optionally add the project polygon layer
- **Forest management plan viewer**: Load management-plan statistics and add a plan-area layer to the map
- **Forest land development search**: Search permit / coordination records by category, applicant, purpose, and municipality
- **Layer helpers**: Add related map layers such as the forest road MVT layer
- **Local cache**: Save acquired API data to the project folder for faster reopening
- **Kozu integration**: Optionally coordinate with `kozu_xml_integrator` when selecting protective forest register rows

## Included tabs

- **保安林台帳**: Parcel-based search and matching against the connected GeoPackage
- **整備事業**: Reserved tab; currently not implemented on the Forest Cloud side
- **森の力**: Search and display project implementation records
- **経営計画**: Load plan summary tables and display plan polygons
- **林地開発**: Search permit / coordination records and optionally add related layers

## Requirements

- QGIS 3.14 or later
  - QGIS 3.20 or later is required to use **Shinrinbo Code Converter** for generating the GeoPackage (full functionality)
- Internet connection for API access and MVT layer access
- A forest planning map layer loaded in QGIS:
  - **Full functionality**: GeoPackage generated by **Shinrinbo Code Converter**
  - **Partial functionality**: Shizuoka Prefecture's official Shapefile distribution (forest stand attributes will be incomplete)
- Workflow limited to **Shizuoka Prefecture**

Tested environment:

- QGIS 3.44.10 on Windows and Linux

## Installation

This plugin is currently intended for **independent ZIP distribution** and is not installed from the QGIS official plugin repository.

1. Download the distributed `fcloud_shizuoka.zip`
2. In QGIS, open **Plugins → Manage and Install Plugins**
3. Choose **Install from ZIP**
4. Select the ZIP file and install
5. Open the plugin from the toolbar icon or **Vector → 静岡県森林クラウド**

## Basic workflow

1. Open a QGIS project and load the target GeoPackage layer
2. Open **Fcloud Shizuoka**
3. Select the connected `.gpkg` layer in the combo box
4. Use each tab to search Forest Cloud records
5. Select table rows to highlight the corresponding features on the map
6. If needed, save the acquired data locally for reuse in the same project

## Cache behavior

Cached data is stored under the project directory in an `fcloud_shizuoka/` folder.

- **保安林台帳**: SQLite cache for full dataset plus Oaza list cache
- **森の力**: SQLite cache by search condition, plus polygon cache
- **経営計画**: SQLite cache plus GeoPackage cache
- **林地開発**: SQLite cache by search condition

The project should be saved before relying on local cache storage.

## Notes

- This plugin is specific to Shizuoka Prefecture data and API endpoints
- Shizuoka Prefecture's official Shapefile distribution can also be used as the source layer, but forest stand attribute data will be incomplete compared to a GeoPackage generated by Shinrinbo Code Converter
- A detailed user manual is included separately as `manual.html`
- The plugin can use `kozu_xml_integrator` when protective forest register rows are selected
